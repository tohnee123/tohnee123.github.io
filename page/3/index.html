如何配置一个稳定的开发环境

准备

我使用 OSX 已经很长一段时间了，OSX 算是比较稳定的操作系统，折腾了几年都没有重装过。但是随着这么多年的使用，以及最近在折腾的各种学习和开发工具，也发现给我的系统带来了一些问题。最近正在考虑重装系统，但是如果继续折腾这些新东西，难免会给新系统带来不稳定的因素。我这次重装完系统后，实在不想再重装系统了。于是我把解决方法放在了虚拟机上。

传统我使用虚拟机主要是解决在 OSX 上运行 Windows 的程序。但是我现在购买的电脑，都是 4 核 8 线程，16G RAM，512G SSD 以上的配置，通过虚拟机的方案，实际上也可以实现开发环境和宿主机的分离。这样我可以在开发环境中尝试新的软件而不至于影响到宿主操作系统的稳定性。

我现在采用的是 VMWare 的虚拟机方案，通过在 VMWare Fusion 上建立 Windows 10、Linux 和 OSX 的虚拟机来减少对宿主操作系统的干扰。采用 VMWare 的一个主要原因是 VMWare 的跨平台的支持能力，而我的电脑有 OSX 和 Windows 两个系统。

环境

我未来相当长一段时间使用如下开发环境。

前端

  1. 开发工具：xcode、HBuilderX 和 Microsoft Azure Notebooks
  2. 开发语言：python、Javascript

后端

  1. 服务主要主要部署在 Linux 上，测试机 CELL181DEV，服务器 CELL603
  2. 服务
    • Hexo + Next：博客工具

配置

Windows 平台

  1. 建立了一个基础的 Windows 安装系统 CELL94（只安装了 Chrome）
  2. 在 CELL194 的基础上建立了一个链接克隆 CELL194_DEV。目前配置了如下软件：
  • Microsoft Visual Studio Code
  • Node Verion Manager for Windows 1.1.7
  • 两个版本的 Node.js

安装

Windows 平台

安装 NVMW

  1. 删除所有的 Node.js 和 npm
  2. 删除 C:\Program Files\nodejs
  3. 下载NVMW 1.1.7
  4. 这是一个 msi 文件，双击安装

备份

  1. 目前主要采用手工备份的方式，在掌握好powershell和Linux shell之后，慢慢过渡到自动备份的方式；

备份设备和目录

  1. CELL004:~/blog//Users/bbgg/celltrix/github/blog/source/*
    • 备份周期：每月备份，每月最后一个周末
    • 备份格式：zip
    • 云端服务：百度云盘
    • 文件名：blogsource_yyyydd
    • 每半年检查一次（检查方法待定）

注意事项

需要注意的是，由于重装系统/回退变得比较容易，那么日常的开发文件的备份需要特别注意。回退之后，开发文件/文档如果没有备份/备份失败，这个影响是沉重的。

如何建立一个类似这样的网站（一）

发表于 2019-12-10 更新于 2019-12-15 分类于 应用程序
本文字数： 2.7k 阅读时长 ≈ 9 分钟

源起

很多年以前，我就想建立一个类似这样的博客网站。大致有如下的要求：

  • 管理方便，能够从 Internet 访问，对手机的支持友好；
  • 费用低廉，我愿意付一定的费用来满足拥有自己博客网站的小心心愿；
  • 内容迁移容易，即使内容再糟糕，也是自己一个个字码起来的。如果主机不得不搬家的话，内容也需要比较容易迁移。最好的形式是可以在本地进行内容的编写，然后上传到博客网站。这个需求以前大部分博客网站支持的都不好，主流是在网站上进行内容的编辑；
  • 界面美观，不用花哨，但要显得专业。之前新浪这些博客的问题是过于花哨，没有美感；
  • 能够方便的支持丰富的文本格式需求，可以支持数学公式和代码格式化；
  • 能够支持留言板功能，方便讨论问题。

回想一下，以前要找到这样一个博客网站，实在是不太容易。直到 Github + Hexo + Markdown + 云服务的出现，实现了上面所有的想法，而且还支持文章目录跳转的功能。现在这些功能都是免费的。真是太棒了！
