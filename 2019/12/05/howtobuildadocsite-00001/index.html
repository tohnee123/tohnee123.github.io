重新出发

Posted on 2019-12-05 Edited on 2019-12-04

对于一个快50岁的人来说，其实学什么东西似乎都显得有点晚。记得30岁那年的春节，我站在武汉的马路上和自己说，要抓紧时间多学习一些东西。可是时间已经流逝了近20年，我自己却没有成长起来。所有人都知道，人和人最大的差别其实不在智商，而在于坚持。可是由于受到自身性格的影响，虽然道理懂，却一直也没有付诸行动。时光流逝，应该是我人生中很大的遗憾。这些年来，我的生活一直很动荡。路是我自己选的，如果我没有选择走这条路，我一定会很遗憾，所以我不后悔我的选择。但是这些年来，总是要顾及生活的需要，心情一直无法沉淀下来，甚至也对自己的梦想没有了追求。这几个月一直孤身在外，没有办法照顾家人，也不能尽子女的责任，心中有愧。当我准备完材料，在这三个星期算是有了那么一些时间，再来规划一下我未来生活该怎么走。

人总是要有一些追求的，目前的状态身体尚可，等待工作，一口饭还能吃的上。对于这个社会的未来，我是悲观的，在这个慢慢走向黑暗的社会里，总是要做一些事情来支持自己继续走下去。于是我又回想起30岁时曾经的追求，重新出发，去寻找自己未完成的梦想。我是喜欢技术的，回顾大学毕业以后，从喜欢技术，到喜欢人文，又回到了技术之中，我感觉更加从容。对于未来，我对自己定义为一个技术爱好者，研究这些一个是消磨时间，一个是希望能够帮助身边正在成长的新的一代。他们担负着比我们更加沉重的期望。

现在的条件，除了自己的脑袋变笨了，其他的条件其实要比过去要好。现在不懂的内容，在百度上一搜，还是可以解决大部分问题；文字看不下去，去bilibili可以看视频；还是搞不掂，还有各种各样的微信群和培训机构，可以足不出户解决学习中大量的交流问题。硬件条件也比以前要好，无论是云服务还是github，无论是开发工具还是硬件设备，对于学习来说都是没有外部理由学不好的，剩下就是自己的努力了。

我的学习目标

我有一个很宏大的学习目标，不知道最后能完成多少。

基础知识

  • 主要是数学，除了高中数学的内容以外，还需要掌握高等数学（微积分）、线性代数和概率论。
  • 英语，这个简直不想多说。

计算机技术

这一部分内容非常多，根据学习的先后次序，大概有如下的内容。

  • Markdown。应该是一个基础的写作工具，我的这个网站的内容都是用 Markdown 完成的。如果有一点点计算机概念的话，应该半天以内就可以掌握主要内容，开始发挥作用。非常好用，最好的特点是，所有的内容都在一个文件中，方便保存；而通过一些服务的支持可以实现非常实用的效果。
  • Hexo 和 Next 模版。准确的说，Hexo 不能称为一门知识，他只是一个博客的框架。但是对于学习来说，我现在需要通过大量的总结和练习（而不是听课）形成自己的理解。而 Hexo 提供了我所梦想的总结工具。我这个网站的目的就是一个自己学习的总结，Hexo 和 Next 模版提供了一切，支持 Markdown、数学公式、章节显示和格式化代码。
  • HTML、CSS 和 JavaScript。我自己对 Web 前端的开发不是特别感兴趣，这三样技术最后形成了 Web 前端开发的基本要求。我现在花时间学习的主要原因一个是为了以后爬虫的学习打下一点基础；另一就是 JavaScript 以后在后台的开发中，是 Node.js 的开发语言，学习 JavaScript 不吃亏。
  • Python。搞计算机的总是要掌握一门语言，我指的掌握是非常熟练的使用，而不是大概知道怎么写的。根据老师的说法，大概总是要写到20000行有效代码，才能大概说自己懂一门计算机语言。学习 Python 的主要原因，不完全是因为这门语言比其他的计算机语言更优秀，而是使用的场景比较多。无论在系统运维、后台开发、科学计算还是 AI 方面，Python 的表现都比较优秀。所以在时间有限的情况下，python 是个性价比比较高的选择。
  • MySQL 和 SQL。SQL 一种数据库查询语言，MySQL 是一个数据库。根据我的需要，我至少需要掌握一门关系型数据库。
  • Linux。学习计算机技术，不知道 Linux 是无法想象的。我希望能够达到 Linux 运维的要求。
  • Shell 和正则表达式。在很多时候，使用编程语言去解决问题，有点像是拿大炮打蚊子，所以还是需要对 Shell 熟练掌握。正则表达式在编程上有着大量的应用场景，好好掌握似乎是必须的选择。
  • 数据结构与算法。数据结构和算法是计算机的基础，这两门课我自己在大学就没有学好，深以为憾。希望现在有机会能够掌握。
  • CCIE，我曾经两次考过 CCNP，可是都没有一鼓作气完成 CCIE。所以 CCIE 的证书成为了我心中的那棵草。如果我之后上班，可能对网络也需要有较深的了解，看时间和缘分吧。

基本的配套工具

  • 一台笔记本电脑，我的是一台13‘的 Macbook Pro。Windows 也没有问题，现在我使用的大部分软件在两个平台上是兼容的。但无论是什么电脑，最后内存都配大一些，用 SSD 的硬盘，时间是值钱的。
  • 一点点钱。因为现在有很多网上现成的云服务，能够很快的帮助我们实现很多功能或者进行实验。条件许可的情况下，试一试这些服务，能给我们带了不一样的学习体验。现在我每年不包括软件，云服务的年开销大概在2000元以下。
  • 需要以下的软件
    • Visual Studio Code（免费）, 可以解决大部分后台和程序设计语言的开发。
    • HBuilderX（免费），一个国产的软件开发工具。这个软件在前端的开发上，我用起来比 VSC（Visual Studio Code）顺手。
    • Dash（免费），一个 Mac 离线的 API 查找工具，大部分常用的 API都可以下载到 Dash 里面。Windows 平台上有 Zeal 和 Velocity，我自己没有用过，在视频里面看到其他人用过。
    • Anaconda（免费），Python 的支持集成包。在 Anaconda 上可以直接下载，有2.x和3.x两个版本，建议使用3.x。2.x后年应该会停止支持了。
    • Node.js（免费），这是一个服务端的开发平台，我现在不会用。但是在发布 Hexo 的时候需要 Node.js 的支持，所以需要安装在自己使用的笔记本电脑上。
  • 需要以下的云服务
    • Github（免费）。在开始的时候，Github 是一个很好的起步。现在大家看到的这个自我学习的总结，我就是使用的 Github webpages 的功能。在未来代码也可以保存在 Github 上。需要注意的是，在国内 Github 访问比较慢。而且由于 Github 可以实现博客的大部分功能，如果 Github 上的内容过于负面，也不知道会不会以后还能访问。所以写的东西最好一定要在自己的机器上有一个备份。
    • 阿里云（开始的时候每年不超过100元）。开始的时候建议用他们的 OSS 云存储服务。如果要省钱的话，建议选择新加坡站点，每个月有5G的存储和流量免费，刚刚开始的时候大部分够用了。
    • 七牛云或者 sm.ms（有一定的免费容量）。由于 Markdown 中不能直接插入图片，所有在 Markdown 中显示的图片都需要放在网络上，通过链接的形式展示。所以需要通过图床的服务。我没有放在阿里云 OSS 上的主要原因是，像 sm.ms 这样的图床服务提供图片预览和代码粘贴的功能，我现在还没有能力写出自己的代码来，所以就暂时寄居别家了。

学习路径

对于我来说，现在虽然时间比较充裕，但是记忆力确实差了许多。很多东西听过就忘记了，所以就想通过这个博客，慢慢总结，形成自己的知识记忆。我目前注意精力分配如下：

  • 基础知识：线性代数的巩固。
  • 电脑技术方面：Python、Web 前端（HTML、CSS 和 JavaScript）、Linux 的使用这三个方面。
