如何建立一个类似这样的网站（二）

发表于 2019-12-15 更新于 2019-12-16 分类于 应用程序
本文字数： 2.2k 阅读时长 ≈ 7 分钟

目标

这一部分，我们的主要目标如下：

  • 安装 Node.js、Hexo 和 Next 主题。这样我们就拥有了一个阶段性成果。
  • 配置 Next 的重点参数。

资源

  • 1. NexT主题配置
  • 2. Hexo语言不生效问题
  • 3. Hexo-NexT (v7.0+) 主题配置

操作

重点步骤

  1. 安装Node.js
  2. 利用 npm 安装 Hexo
  3. 安装 Next 主题

安装 Hexo

配置 Hexo

  1. 更改 Hexo 的默认主题。我不喜欢 Hexo 自带的主题，选择了现在比较流行的 Next 主题，版本是 Next 7.6.0。这个版本配置文件有一些更改，但是大部分前面提供的资源内容还是可以用在这个版本上；
  2. 可以直接从 Github 上安装 Next 7.6.0。安装命令如下：git clone --branch v7.6.0 https://github.com/theme-next/hexo-theme-next themes/next。指令要在主目录下键入；
  3. 修改博客主目录下的 _config.yml 文件内容。

发布流程

一般我们都会使用 hexo new <title> 来建立文章，这条指令会将新文章建立在 source/_posts 目录下，当使用 hexo generate 编译 markdown 文件时，会将其 HTML 结果编译在 public 目录下，之后 hexo deploy 将会把 public 目录下所有文章部署到 GitHub，这是我们熟悉的 Hexo 流程。

增加文章

增加一篇新的文章在博客的主目录下输入指令hexo n "文章名"就可以了。新的 Markdown 文件会产生在source/_posts目录下。

发布文章

发布写好后的文章需要两条指令。

访问文章

发布好之后，大概还需要 5～10分钟的时间，就可以通过浏览器访问你建立的博客站点了。我的博客站点是 http://tohnee123.github.io/

其他

有的时候，想在发布到网上之前，先在本地看看文章的内容。可以输入如下指令。

Hexo 插件

  • 查看已安装的 Hexo 插件

配置 Next

我选用的 Next 主题的版本是 7.6.0，也可以在 github 上下载下载 Next 指定版本。下载的 zip 文件解压后，复制到 theme 下就可以。注意要把目录名改为 next。

配置文件

主要的 Next 配置文件如下：

  1. themes/next 目录下的 _config.yml；
  2. themes/next/languages 目录下的语言文件；

更改语言

将 themes/next/languages 目录下的 zh-CN.yml 更名为 zh-Hans.yml

更改风格

Next 提供了四种不同的风格，我个人比较喜欢 Gemini 风格。需要修改 Next 的 _config.yml。找到 Schemes 这个 Section，修改如下，启用 Gemini 风格。

标签和类别页面

增加标签和类别页面。默认情况下这两个是没有页面链接的，所以需要按照资源3的建议增加。

注意事项

  • 安装Node.js。目前我是从 node.js 网站上直接下载 OSX 版本的二进制文件直接安装的。在网上还有另外两种办法，一种是通过 homebrew，一种是通过 crul，这两种办法现在我没有试过，也不知道通过多种渠道安装不同版本的 node.js 会有什么后果。
  • 目录结构，安装完 Hexo 后，会在安装目录下有若干子目录或者文件，比较重要的有：
    • _config.yml：Hexo 的整体配置，比如语言，采用的主题、git 的仓库等内容；
    • themes：可以安装各种主题；
    • 主目录下的 source/_posts：我们准备发布到网络上的各种 Markdown 文章；
    • 主目录下的 source/_drafts：这个目录下的文件不会被发布到网络上。
